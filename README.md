# Spine Registration Framework for CT-Derived Vertebral Models



This repository contains the source code developed for my Master's Thesis on the generation, deformation, and registration of patient-specific spinal models reconstructed from CT images.



The project provides a complete pipeline that includes:



\- Generation of 3D vertebral meshes from CT images.

\- Adaptation of CT-derived models to an OpenSim anatomical model.

\- Synthetic dataset generation with controlled vertebral deformations.

\- Evaluation of classical rigid and non-rigid registration algorithms.

\- Deep-learning registration using LePard.

\- Post-processing and refinement of LePard predictions.

\- Quantitative evaluation and comparison of registration methods.

\# Workflow



The complete workflow consists of four main stages.



\## 1. Mesh Generation



Patient-specific vertebral meshes are reconstructed from segmented CT images using Marching Cubes.



Main scripts:



\- `Mesh\_generation.py`

\- `new\_spine.py`



Outputs:



\- Individual vertebral meshes

\- Complete spine mesh

\- Separator indices

\- Scaling factor for OpenSim alignment



\---



\## 2. Dataset Generation



Synthetic vertebral configurations are automatically generated by applying controlled rotations to each vertebra.



Main scripts:



\- `generate\_configuration.py`

\- `generate\_dataset.py`

\- `Compute\_yaml\_parameters.py`



Outputs:



\- Fixed and moving point clouds

\- Motion fields

\- LePard-compatible dataset

\- YAML parameters



\---



\## 3. Registration Algorithms



Several registration approaches are implemented and compared.



\### Rigid Registration



\- FPFH feature extraction

\- RANSAC

\- ICP refinement



Script:



\- `Rigid\_ICP.py`



\### Non-Rigid Registration



Implemented algorithms:



\- CPD

\- GMMReg

\- SVR



Scripts:



\- `NonRigid\_CPD.py`

\- `GMMReg.py`

\- `SVR.py`



Each algorithm computes quantitative registration metrics that are automatically saved in the `Results` folder.



\---



\## 4. LePard Post-processing



After LePard inference, several post-processing steps are performed:



\- deformation propagation

\- prediction conversion

\- N-ICP refinement

\- high-resolution reconstruction

\- computation of execution times

\- final evaluation metrics



Scripts:



\- `med\_res\_post\_clouds\_generator.py`

\- `NPZ\_to\_MAT.py`

\- `N\_ICP\_refinement.m`

\- `High\_resolution\_post\_NICP.py`

\- `Total\_time.py`

\- `Lepard\_results\_generator.py`



\---



\# Evaluation



The framework evaluates different registration methods using metrics such as:



\- Global Registration Error

\- Point-to-Point Distance

\- Distance Preservation Error

\- Overlap

\- Execution Time



Boxplots comparing all methods can be generated using:



```

Box\_plots.py

```
